# Бункер — Технічна специфікація гри

> Фінальні рішення для реалізації. Всі спірні питання вирішені.

---

## 1. Суть і мета

Постапокаліптична дискусійна гра на виживання. Сталася глобальна катастрофа, є бункер з обмеженою кількістю місць. Гравці мають довести свою корисність і переконати інших вигнати слабших.

**Місць у бункері:** `Math.floor(players.length / 2)` — рівно половина.  
**Гравці:** 4–15 осіб.  
**Тривалість партії:** ~30–60 хвилин.

---

## 2. Параметри гравця

Кожен гравець отримує унікальний набір із 6 атрибутів + 1-2 карти дій.

```js
player = {
    id:         0,           // індекс у кімнаті
    name:       'Богдан',
    isAlive:    true,
    isSilenced: false,       // "Тіньовий бан" — не може писати в чат 1 раунд
    immunityRounds: 0,       // кількість раундів імунітету від вигнання
    attributes: {
        profession: { value: 'Зварювальник', isRevealed: false },
        biology:    { value: 'Чоловік, 34 роки, плідний', isRevealed: false },
        health:     { value: 'Астма', isRevealed: false },
        hobby:      { value: 'Самогоноваріння', isRevealed: false },
        trait:      { value: 'Параноїк', isRevealed: false },
        baggage:    { value: 'Аптечка першої допомоги', isRevealed: false },
    },
    actionCards: [
        { id: 'act_luz', name: 'Ой, у лузі...', used: false }
    ]
}
```

**Свої атрибути** гравець бачить всі завжди.  
**Чужі атрибути** — тільки де `isRevealed: true`.  
**Чужі карти дій** — назва картки, але не опис (таємниця до застосування).

## 3. Фази гри

```
LOBBY → SCENARIO_PICK → GAME_START → [ROUND_REVEAL → DISCUSSION → VOTING] × N → END_GAME
```

### 3.0 LOBBY
Стандартна зала очікування. Хост бачить кнопку старту при ≥ 4 гравцях.

### 3.1 SCENARIO_PICK (15–30с)
Тільки хост. Вибирає сценарій зі списку 10 готових.  
Всі бачать: **назву** і **опис катастрофи** (але ще не бункер — інтрига).  
Хост підтверджує → перехід до GAME_START.

### 3.2 GAME_START (60с)
Всім показується:
- Повний опис катастрофи
- Характеристики бункера (площа, запаси, проблема)
- Кількість місць: N з M

Кожен бачить **тільки свої** атрибути. Час на ознайомлення — 60 секунд, після чого автоматичний перехід.

### 3.3 ROUND_REVEAL

Кількість раундів = `players.length - bunkerCapacity - 1` (вибувають по одному, останній раунд перевіряє умову).

| Раунд | Правило |
|-------|---------|
| 1 | Всі **обов'язково** розкривають **Професію** |
| 2 | Всі **обов'язково** розкривають **Біологію** |
| 3+ | Кожен **сам обирає** який атрибут розкрити (Здоров'я, Хобі або Рису) |

**Механіка розкриття:**
- Всі вибирають **одночасно** (не по черзі)
- Таймер: **30 секунд** на вибір
- Якщо не вибрав → сервер автоматично розкриває перший нерозкритий атрибут
- Коли всі підтвердили → атрибути з'являються на екранах з CSS-flip анімацією

**Застосування карт дій** можливе до того як натиснути "Підтвердити" у фазі розкриття.

### 3.4 DISCUSSION (120с)
- Відкритий чат для всіх живих гравців
- Вибулі — **читають, але не пишуть** (`isSilenced` по факту вигнання)
- Таймер 120 секунд, після — автоматичний перехід до VOTING
- **Карти дій** можна застосовувати протягом всього обговорення
- Виняток: "Спростування" — тільки під час VOTING

### 3.5 VOTING (60с)

**Механіка:**
- Кожен живий гравець голосує **проти одного** іншого (не можна за себе)
- Таймер 60 секунд; не проголосував → голос не зараховується
- Голосування **відкрите** (всі бачать хто проти кого)
- Виняток: карта "Блекаут" робить поточне голосування таємним

**Підрахунок:**
- Гравець з найбільшою кількістю голосів вибуває
- **Нічия** → проводиться **повторне голосування** між тими хто в нічиї (30с)
- Якщо нічия повторилась → **випадковий** вибір між ними (показуємо "жереб")
- Вибулий гравець: всі його атрибути розкриваються повністю, стає глядачем

**Карта "Спростування"** грається в момент оголошення результату (до видалення гравця) і дає ще 1 раунд.

### 3.6 ROUND_END_CHECK
Після кожного вигнання сервер перевіряє:
- `livingPlayers.length === bunkerCapacity` → **END_GAME**
- Інакше → новий раунд (ROUND_REVEAL)

### 3.7 END_GAME
- Всі атрибути всіх гравців розкриваються
- Показується список тих хто вижив
- **Епілог** (2-3 речення) — генерується шаблонно на основі комбінації вижилих або через Gemini API
- Кнопки: "Реванш" (хост) і "Нова гра"

## 4. Карти дій — коли можна грати

| Карта | Фаза |
|-------|------|
| Ой, у лузі... | DISCUSSION або VOTING |
| Люстрація | ROUND_REVEAL або DISCUSSION |
| Хабар | DISCUSSION |
| Друге дихання | DISCUSSION |
| Перекваліфікація | DISCUSSION |
| Тіньовий бан | DISCUSSION |
| Бавовна | DISCUSSION |
| Гуманітарна допомога | DISCUSSION |
| Воєнний стан | VOTING (до підрахунку) |
| Кумівство | VOTING (до підрахунку) |
| Суворий карантин | DISCUSSION |
| Медична реформа | DISCUSSION |
| Блекаут | VOTING (до підрахунку) |
| Депортація | DISCUSSION |
| Донат на ЗСУ | DISCUSSION |
| Націоналізація | DISCUSSION |
| Спростування | VOTING (після оголошення результату) |

**Кожна карта використовується 1 раз** (`used: true` після застосування).

## 5. Sanitize-логіка (що бачить кожен гравець)

```js
function sanitizeBunker(state, forIdx) {
    return {
        gameType:       'bunker',
        phase:          state.phase,
        round:          state.round,
        scenario:       state.scenario,        // публічний
        bunkerCapacity: state.bunkerCapacity,
        timeDeadline:   state.timeDeadline,
        players: state.players.map((p, i) => ({
            id:      p.id,
            name:    p.name,
            isAlive: p.isAlive,
            // Своя картка — всі атрибути, чужа — тільки розкриті
            attributes: i === forIdx
                ? p.attributes
                : Object.fromEntries(
                    Object.entries(p.attributes).map(([k, v]) => [
                        k, v.isRevealed ? v : { value: '???', isRevealed: false }
                    ])
                ),
            // Карти дій: свої — повністю, чужі — тільки кількість і назва
            actionCards: i === forIdx
                ? p.actionCards
                : p.actionCards.map(c => ({ id: c.id, name: c.name, used: c.used })),
            isSilenced:     p.isSilenced,
            immunityRounds: p.immunityRounds,
            // Чи вже розкрив атрибут у поточному раунді розкриття
            hasRevealed: p.hasRevealed || false,
        })),
        votes:    state.phase === 'voting' ? state.votes : {},  // відкрите голосування
        myId:     forIdx,
        log:      state.log.slice(0, 30),
    };
}
```


## 7. Таймери по фазах

| Фаза | Таймер | Дія при вичерпанні |
|------|--------|--------------------|
| SCENARIO_PICK | — | Хост обирає вручну |
| GAME_START | 60с | Автоматичний перехід до ROUND_REVEAL |
| ROUND_REVEAL | 30с | Автоматичне розкриття першого нерозкритого атрибуту |
| DISCUSSION | 120с | Перехід до VOTING |
| VOTING | 60с | Підрахунок голосів (не проголосували — не впливають) |
| Повторне VOTING | 30с | Жереб між тими хто в нічиї |

## 8. Технічна реалізація в проекті

### Файли
```
server.js                      ← додати createBunkerState, sanitizeBunker, processBunkerAction
public/games/bunker/
    bunker.js                  ← клієнтський рендер
    bunker.css                 ← стилі (темна тематика, атомний бункер)
    scenarios.js               ← масив 10 сценаріїв (окремий файл)
    attributes.js              ← бази даних атрибутів
public/index.html              ← #bunker-screen + game-card
```

### Паттерни з існуючих ігор
- **Sanitize** → за зразком `sanitizeTysyacha` / `sanitizeDurak`
- **Таймери** → за зразком `dStartTurnTimer` (Дурак)
- **Чат** → той самий `chatMessage` event що і в Тисячі/Дураку
- **Голосування** → за зразком `processMafiaAction` (voting логіка)
- **Лог** → `addLog(state, text)` — вже є

### Специфічне для Бункера
- CSS `transform: rotateY(180deg)` — фліп картки при розкритті атрибуту
- Локальні маркери 🟢🔴 — `localStorage` на клієнті, не передаються серверу
- Картка гравця (досьє) — grid layout, 2-3 колонки залежно від кількості гравців
